# Простой банк для изучения Go backend разработки

Данное приложение представляет собой простой банк, разработанный для изучения backend-разработки на Go.

## Основные функции

1. **Создание и управление аккаунтами:** регистрация пользователей с указанием имени владельца, баланса, валюты и других параметров.
2. **Учёт изменений баланса:** фиксация всех операций, влияющих на баланс счёта.
3. **Денежные переводы между счетами:** выполнение транзакций между аккаунтами.

- **github.com/spf13/viper**: для управления конфигурацией приложения.

## Техническая реализация денежных переводов

```go
type Store struct {
    *Queries
    db *sql.DB
}
```

Объединяет:

- Сгенерированные SQL-запросы (Queries)
- Подключение к базе данных
- Механизм выполнения транзакций

### Транзакционный механизм

Метод `execTx` обрабатывает жизненный цикл транзакции:

1. Начало транзакции с `BeginTx`
2. Выполнение callback-функции в контексте транзакции
3. Автоматический откат при ошибках
4. Фиксация изменений при успешном выполнении

```go
func (store *Store) execTx(ctx context.Context, fn func(*Queries) error) error {
    tx, err := store.db.BeginTx(ctx, nil)
    // ... обработка ошибок ...
    q := New(tx)
    err = fn(q)
    // ... commit/rollback ...
}
```

### Логика денежного перевода (TransferTx)

**Последовательность операций:**

1. Создание записи о переводе

```go
result.Transfer, err = q.CreateTransfer(ctx, CreateTransferParams(arg))
```

2. Создание двух записей в истории операций:

- Дебетовая операция (списание)

```go
result.FromEntry, err = q.CreateEntry(ctx, CreateEntryParams{
    AccountID: arg.FromAccountID,
    Amount:    -arg.Amount,
})
```

- Кредитовая операция (зачисление)

```go
result.ToEntry, err = q.CreateEntry(ctx, CreateEntryParams{
    AccountID: arg.ToAccountID,
    Amount:    arg.Amount,
})
```

3. Атомарное обновление балансов:

```go
if arg.FromAccountID < arg.ToAccountID {
    result.FromAccount, result.ToAccount, err = addMoney(...)
} else {
    result.ToAccount, result.FromAccount, err = addMoney(...)
}
```

### Защита от взаимных блокировок (Deadlock Prevention)

**Ключевые механизмы:**

1. **Строгий порядок блокировок**:
   - Всегда обновляем счета в порядке возрастания их ID
   - Предотвращает циклические зависимости блокировок

2. **Атомарные операции UPDATE**:

```sql
UPDATE accounts
SET balance = balance + $1
WHERE id = $2
RETURNING *;
```

- Вычисление нового баланса на уровне БД
- Исключает race condition между параллельными запросами

3. **Изоляция транзакций**:
   - Уровень изоляции по умолчанию (обычно READ COMMITTED)
   - Гарантирует целостность данных при параллельном доступе

### Функция addMoney

```go
func addMoney(...) {
    account1 = q.AddAccountBalance(...) // Первое обновление
    account2 = q.AddAccountBalance(...) // Второе обновление
}
```

Особенности:

- Всегда выполняется в строгом порядке (по ID счетов)
- Возвращает актуальные данные счетов после обновления
- Обрабатывает ошибки на каждом этапе

### Архитектурные преимущества

1. **Идемпотентность** - повторные вызовы безопасны
2. **Атомарность** - все операции выполняются как единое целое
3. **Масштабируемость** - паттерн применим для сложных бизнес-процессов
4. **Надёжность** - автоматическое восстановление при сбоях
